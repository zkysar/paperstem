Flow 1 + 2

During upload — per-import checkbox

The toggle lives in the UploadDrawer, not in global settings. Default remembered per user via localStorage. Classification runs concurrently with the upload (browser decodes the file, runs YAMNet + chroma in parallel — done in <15 s for a 90-min file).

Flow 3

Project view — reviewing auto-sections

After upload, the user lands on the project page. The timeline shows detected sections. Auto sections have a dashed border, faded color, and a confidence chip. Two confidence tiers: confident (slate) and tentative (warm yellow).

Visual decisions in this mock:
  • Dashed border + slate color for auto, vs solid + accent-orange for manual.
  • Italic label on auto sections to reinforce "not committed yet."
  • Small ● dot in slate marks an auto section at a glance (visible even when zoomed out).
  • Chip shows confidence as a percent when high, "low" or "auto" otherwise.
  • Tentative variant (yellow dashed) for low-confidence matches. Distinct so the user knows to scrutinize harder.
  • Toolbar shows count and bulk actions only when there are auto sections present.
Flow 4

Clicking an auto-section — accept / rename / discard

Click an auto section to open the popover. Accept promotes it to a normal manual section. Rename (or moving a boundary) also flips it to manual — any deliberate touch counts as acceptance.

Flow 2 (alt)

Toolbar — during classification

While YAMNet + chroma run in the browser (parallel to the upload), the toolbar shows a non-blocking progress indicator. The user can keep using the rest of the UI; if they navigate away, classification continues and sections appear on return.

Flow 7

Project list — CLI-imported projects with pending auto-sections

When the Tascam Model 12 importer runs auto-classification (the --auto-classify flag), the user wasn't in the web app at that moment. Next time they open the band view, a slate badge tells them which projects have suggestions waiting.
